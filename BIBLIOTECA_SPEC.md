# Biblioteca — Project Spec

**Biblioteca** ("library," in the knowledge-collection sense) is an interactive dashboard that visualizes your Claude Code skills/tools library, recommends the best one for a given task, and surfaces session/config health info. It's designed to be simple, dependency-light, and easy to keep current.

---

## Goals

1. Display all Claude Code skills and tools as an interactive node graph.
2. Accept a task description as input and recommend the best-matching skill/tool, with ranked results and reasoning.
3. Surface a "Session Health" panel showing configured hooks, installed plugins, and plugin-specific safe-exit/recovery info.
4. Match the visual theme/branding pulled from the **Ava core HTML file**.
5. Stay easy to run and easy to update — no build step, single-file where possible.

---

## Architecture Decisions

| Area | Decision | Rationale |
|---|---|---|
| Recommendation engine | **Groq API (free tier) — `llama-3.1-8b-instant`** | Quick setup (just an API key, no local install), most permissive free-tier limits of Groq's lineup, and zero VRAM contention with Ava since it's not touching the GPU. Model choice can be confirmed against `console.groq.com/docs/models` at build time, since Groq's exact lineup shifts periodically. |
| Node graph rendering | **vis-network** or **force-graph**, loaded via CDN | No build tooling required. |
| Build approach | **Single HTML/JS/CSS file**, run via `python -m http.server` or similar | Simplest runnable setup; avoids CORS issues from opening `file://` directly. |
| API key storage | **Gitignored local config file** | Never hardcoded in HTML/JS, even for local-only use. |
| Skills/tools index | **JSON file, regenerated by a rescan script** | Keeps the dashboard in sync as skills are added without manual edits. |
| Visual theme | **Pulled directly from the Ava core HTML file** | Claude Code should reference that file's actual markup/CSS directly rather than re-specifying color palette/fonts/layout manually. |

## Core Features

### 1. Skills/Tools Node Graph
- Scans Claude Code skills/tools directories and generates a JSON index: `name`, `description`, `category`, `trigger keywords`.
- Renders as an interactive graph (vis-network or force-graph), nodes connected by shared category or overlapping use-case.
- Filterable/searchable.

### 2. Task Recommendation
- Text input for a task description (e.g. "build a website").
- Calls the Groq API (free tier, model: `llama-3.1-8b-instant`) to rank the best-matching skill/tool against the JSON index.
- Highlights the recommended node in the graph and displays the reasoning.

### 3. Session Health Panel
- Reads `.claude/settings.json` and the plugins directory to list **configured** hooks (SessionStart, Stop, SessionEnd, etc.) and installed plugins.
- **Diagnostic/informational only** — reads config at scan time, does not monitor a live running session (that would require a background daemon, a separate/heavier project).
- Detects known plugins with background workers or local SQLite state (e.g. claude-mem, claude-brain) via their config/marker files.
- If a known plugin is detected, surfaces its **actual documented repair command** (e.g. `npx claude-mem repair`) — never a generic "wipe lockfiles" fix.
- If no known plugin is detected, shows a note that recovery steps are plugin-specific, with a link out rather than guessing.
- Includes a short "safe exit" reference card:
  - Use `/exit` rather than force-killing the terminal.
  - Avoid killing background worker processes mid-write.
  - Avoid rapid `/clear` + immediate relaunch loops while a background worker is active.

### 4. Rescan Script
- Regenerates the skills/tools JSON index on demand as new skills are added.
- Optionally regenerates the Session Health config snapshot at the same time.

### 5. Launcher Script (Windows)
- A `launch-biblioteca.bat` file in the same folder as the dashboard, so it can be run with a double-click — no commands to remember.
- Behavior: switches to its own folder, opens the default browser to `http://localhost:8000`, then starts `python -m http.server 8000`.
- Closing the terminal window it opens stops the server.
- Optionally turned into a desktop shortcut (with a custom icon) for one-click access.

## Data Structure Notes

- Skills/tools index and Session Health data should likely be **separate JSON files** — one is a capability catalog, the other is config/diagnostic state, and they update on different triggers (rescan vs. every load).
